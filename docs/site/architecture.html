Architecture

How BitAgent is structured — layers, entry points, and module responsibilities.

System overview

BitAgent is a FastAPI application deployed on Railway. All agents share a single process, a single LNbits wallet client, and a single Nostr identity. Agents are FastAPI routers mounted under path prefixes.

Layer breakdown

Entry Points

main.py

Railway / cloud deployment. Mounts all routers, handles CORS, runs Nostr broadcast on startup.

  • mcp_server.py — MCP stdio for Claude
  • start9_server.py — preserved, do not modify
Agents — src/agents/

FastAPI routers

Each agent is a self-contained package with its own payment logic. No shared decorator; each checks payment_hash inline.

  • polyglot_agent/
  • coordinator_agent/
  • price_oracle_agent/
  • web_fetch_agent/
  • search_agent/
  • streamfinder/ (A2A only)
  • identity_agent/ (A2A only)
Network — src/network/

Discovery + Registry

P2P discovery over DHT and Nostr. Agent registry endpoint exposes known agents over HTTP.

  • p2p_discovery.py — DHTNode, EnhancedNostrDiscovery, P2PDiscoveryManager
  • registry_router.py — GET /agents/registry
  • nostr.py — Kind 30078 announcements
Wallets

Payment backends

Two tiers: Lightning via LNbits, ecash via Fedimint. AgentWallet is lazy (raises on missing config). FedimintWallet is eager but harmless when unconfigured.

  • agent_wallet.py — LNbits REST wrapper
  • src/wallets/fedimint_wallet.py — ecash client
  • lnbits_client.py — raw HTTP
Security — src/security/

Auth + Encryption

JWT auth, API key hierarchy, AES-256-GCM / ChaCha20-Poly1305 encryption, input sanitisation, Pydantic models at boundaries.

  • authentication.py
  • encryption.py
  • secure_endpoints.py
  • payment_security.py
  • secure_communication.py
Identity — src/identity/

DID + NIP-05

DID document creation for four methods. NIP-05 registration and trust scoring backed by SQLite.

  • enhanced_did.py — did:key/web/bitcoin/nostr
  • did.py — basic DID operations
  • identity_agent/store.py — SQLite

Wallet lazy-loading pattern

All live agents follow this pattern so that missing credentials don't crash startup — payments are simply disabled in dev mode.

from src.wallets.fedimint_wallet import FedimintWallet

_fedimint = FedimintWallet()   # eager — safe, no required args
_wallet = None                  # lazy — raises if LNBITS_* not set

def _get_wallet():
    global _wallet
    if _wallet is None:
        try:
            from agent_wallet import AgentWallet
            _wallet = AgentWallet()
        except Exception:
            pass
    return _wallet

MCP server

Claude can call BitAgent agents as tools via the MCP stdio server. No HTTP — agent modules are imported directly. No payment gate in MCP mode; cost is reported in the response but not enforced.

# .mcp.json (Claude Code config)
{
  "mcpServers": {
    "bitagent": {
      "command": "python",
      "args": ["mcp_server.py"]
    }
  }
}

Available MCP tools: translate, search, fetch_url, get_price, convert_sats. Each returns a cost_sats field.

SDEN relationship

SDEN (Sensor Data Exchange Node) is a companion protocol for IoT devices that sell signed sensor data via Lightning. A SDEN producer node is a BitAgent agent — it uses BitAgent's wallet, Nostr discovery, and DID identity. The repos are one system; SensorAgent should extend BitAgent's Agent base class.
