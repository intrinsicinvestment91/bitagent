Agents

Seven live agents, each autonomous, each accepting Lightning or ecash payments.

🌐 PolyglotAgent

● live 100–250 sats /polyglot

Translates text between languages and transcribes audio files using OpenAI Whisper. The cheapest and most-used agent in the stack — translation drives the coordinator and MCP tool integrations.

Services

  • translate — 100 sats/request
  • transcribe — 250 sats/request (Whisper, local-only)

Notes

  • Whisper excluded from prod build (avoids PyTorch ~2 GB pull)
  • Translation is live in production
  • Accepts Lightning and Fedimint ecash
# Translate English → Spanish
curl -X POST https://bitagent-production.up.railway.app/polyglot/translate \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Hello, world",
    "source_lang": "en",
    "target_lang": "es",
    "payment_hash": "<paid_hash>"
  }'

🎯 CoordinatorAgent

● live 350 sats /coordinator

Orchestrates multi-step workflows by chaining calls to specialist agents. Accepts a high-level task and returns a combined result — the client pays once; the coordinator handles all downstream agent calls.

Capabilities

  • Sequential agent chaining
  • Result aggregation
  • Internal retry on agent failure

Typical workflow

  • Search → summarise → translate
  • Fetch page → extract → translate
  • Price → convert → format
# Chain: search + translate in one call
POST /coordinator/coordinate
{
  "task": "search for bitcoin news and translate to German",
  "payment_hash": "<paid_hash>"
}

📈 PriceOracleAgent

● live 2 sats /oracle A2A: oracle.*

Returns real-time crypto prices at 2 sats per request — the cheapest paid agent in the stack. CoinGecko is the primary source with Binance as a fallback. Results are cached for 60 seconds to avoid hammering upstream APIs.

A2A methods

  • oracle.price — single asset
  • oracle.prices — multiple assets
  • oracle.convert — cross-asset conversion

Sources

  • Primary: CoinGecko
  • Fallback: Binance ticker
  • Cache TTL: 60 seconds
# Get BTC price in USD
POST /a2a
{
  "method": "oracle.price",
  "params": { "asset": "bitcoin", "currency": "usd" }
}

🌍 WebFetchAgent

● live 25 sats /fetch A2A: fetch.url

Fetches web pages and returns their content. Includes SSRF protection that blocks requests to private IP ranges (RFC 1918, loopback, link-local) to prevent internal network access.

Security

  • Blocks 10.x.x.x, 172.16–31.x.x, 192.168.x.x
  • Blocks 127.x.x.x and ::1
  • Blocks 169.254.x.x (link-local)

A2A method

  • fetch.url — returns page text/html
  • Configurable timeout
  • User-agent spoofing supported
# Fetch a URL
POST /a2a
{
  "method": "fetch.url",
  "params": { "url": "https://example.com" }
}

🔎 SearchAgent

● live 10 sats /search A2A: search.query

Runs web searches with a three-tier fallback chain. Brave Search is tried first (fastest, privacy-respecting), then SearXNG (self-hosted meta-search), then DuckDuckGo via ddgs as a last resort.

Fallback chain

  • 1. Brave Search API
  • 2. SearXNG instance
  • 3. DuckDuckGo (ddgs library)

MCP tool

  • Exposed as Claude tool via MCP server
  • No HTTP — direct module import
  • Cost reported per call in sats
# Search the web
POST /a2a
{
  "method": "search.query",
  "params": { "query": "lightning network capacity 2025" }
}

📡 StreamfinderAgent

● live 100 sats A2A only

The reference implementation of the Agent-to-Agent JSON-RPC pattern. Provides a streamfinder.search method that can be optionally gated behind NIP-05 identity verification and trust scoring.

A2A method

  • streamfinder.search — query stream DB
  • Optional identity trust gate
  • Requires requester_pubkey when gate active

Identity gate (opt-in)

  • IDENTITY_REQUIRE_FOR_PAID_SERVICES=true
  • IDENTITY_MIN_TRUST_SCORE=0.25 (default)
  • IDENTITY_FREE_QUERIES=false
# Search with identity gate enabled
POST /a2a
{
  "method": "streamfinder.search",
  "params": {
    "query": "bitcoin price feed",
    "requester_pubkey": "<nostr_pubkey_hex>"
  }
}

🪪 IdentityAgent

● live 10–1000 sats A2A only

Handles NIP-05 identity registration (name@domain → Nostr pubkey mapping) and maintains a trust score for each registered pubkey. Backing store is SQLite. Exposes the /.well-known/nostr.json endpoint required by the NIP-05 spec.

A2A methods

  • identity.register_nip05
  • identity.get_identity
  • identity.list_verified
  • identity.search
  • identity.get_trust_signal

Trust scoring

  • Score range: 0.0 – 1.0
  • Factors: payment history, age, activity
  • Used as gate for streamfinder.search
  • Deterministic (same pubkey → same signal)
# Register a NIP-05 handle
POST /a2a
{
  "method": "identity.register_nip05",
  "params": {
    "handle": "alice",
    "domain": "bitagent-production.up.railway.app",
    "pubkey": "<nostr_pubkey_hex>"
  }
}
