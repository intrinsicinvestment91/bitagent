Payment pattern

All paid endpoints follow the same 402 → pay → resubmit flow. If payment_hash is absent, the response is HTTP 402 with an invoice.

FieldDirectionDescription
payment_hashRequestSHA-256 of the paid invoice preimage (hex)
ecash_notesRequestFedimint ecash notes string (alternative to payment_hash)
invoice402 Responsebolt11 Lightning invoice
amount_sats402 ResponseAmount required in satoshis
cost_sats200 ResponseAmount deducted (echoed back to client)

Polyglot — /polyglot

POST /polyglot/translate 100 sats
FieldDescription
textrequiredText to translate (max 5000 chars)
source_langrequiredBCP-47 language code (e.g. "en")
target_langrequiredBCP-47 language code (e.g. "es")
payment_hashoptionalOmit to receive invoice
 
# Request
{
  "text": "Hello, world",
  "source_lang": "en",
  "target_lang": "es",
  "payment_hash": "1b741d85..."
}

# 200 Response
{ "translated_text": "Hola, mundo", "cost_sats": 100 }

A2A JSON-RPC — POST /a2a

All A2A methods share a single endpoint using JSON-RPC 2.0 envelope.

{
  "jsonrpc": "2.0",
  "method": "<method>",
  "params": { ... },
  "id": 1
}

# Response
{ "jsonrpc": "2.0", "result": { ... }, "id": 1 }

oracle.*

MethodParamsReturns
oracle.priceasset, currency?price_usd, timestamp
oracle.pricesassets[], currency?prices{}
oracle.convertamount, from_asset, to_assetconverted_amount

search.*

MethodParamsReturns
search.queryquery, max_results?results[]

fetch.*

MethodParamsReturns
fetch.urlurl, timeout?content, status_code

identity.*

MethodParamsReturns
identity.register_nip05handle, domain, pubkey, relays?invoice or success
identity.get_identityhandle, domainidentity record
identity.list_verifieddomain?verified[]
identity.searchquerymatches[]
identity.get_trust_signalpubkeytrust_score, warning?

streamfinder.*

MethodParamsReturns
streamfinder.searchquery, requester_pubkey?streams[]
